                                              Jason Watmore Mod Omar • 10 months ago

                                              Hi Omar,

                                              The cookie is stored only while you're logged in to the application and is cleared when you log out, so the only way someone would be able to access it is if they get access to your machine while you're still logged in, in which case they would have access to the application regardless of whether the cookie is there or not.

                                              That being said the only purpose of cookie is to enable the user to stay logged in between browser refreshes and sessions. If this isn't a requirement you could remove this functionality.

                                              With Basic HTTP Authentication the username and password are sent with every request to the api so even without cookies you need to use SSL encryption to secure your application.

                                              Cheers,
                                              Jason

                                                                  Jason Watmore Mod Mandar Chitale • a year ago

                                                                  Hi Mandar,

                                                                  Switching to token based authentication won't require too many changes to this example, the main thing will be changing the "Authorization" HTTP header value from "Basic [AUTH DATA]" to "Bearer [TOKEN]".

                                                                  The token should be returned by the web service on successful authentication, if you update the example to use this token in the http auth header instead of the base64 encoded username and password (authdata), everything should work.

                                                                  I've been meaning to get around to posting a JWT example as well, but in the mean time here is a good Angular JWT example that uses NodeJS on the backend - https://auth0.com/blog/2014/01...

                                                                  Cheers,

                                                                                                                                                Hi Agung, make sure your server is returning a with http status 200 and the following in the response body:

                                                                                                                                                { success: false, message: 'Username or password is incorrect' }

                                                                                                                                                If your server is returning a 400 status you'll need to add a ".error()" callback to your http post request, eg:

                                                                                                                                                $http.post('/api/authenticate', { username: username, password: password })
	.success(function (response) {
		callback(response);
	})
	.error(function (response) {
		callback(response);
	});
